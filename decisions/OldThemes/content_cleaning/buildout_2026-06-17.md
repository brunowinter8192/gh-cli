# Content Cleaning Buildout — index_discussions + index_issues (2026-06-17)

Process record for the noise-stripping added to both RAG-indexing pipelines this session. Final state lives in `decisions/discussion_indexing.md`, `decisions/issue_indexing.md`, `src/github/DOCS.md`. This file = the journey, the rejected alternatives, the domain knowledge.

## Trigger

`rag-cli index --collection github_discussions` aborted: **15 of 78** discussion threads silently un-indexed. Cause: embedding server returns HTTP 400 — a **2405-token chunk exceeds the 2048-token context** (`-c 2048`), and the whole run aborts. Root cause = **dosu-bot footers**: shields.io badge URLs + camo-proxied image URLs render as **2000–6299 character runs with no spaces**, which the char-based recursive chunker (2000 chars, splits down to ` `) cannot split → emits one oversized chunk.

Reframed (correctly, by user) as a **gh-cli cleaning gap**, not rag-cli: `strip_discussion_noise()` only did `[ANSWER]`-dedup / title / metadata / token-redaction — zero image/badge/bot handling. The original `index_discussions` probe ran on a small clean repo (`gastownhall/beads`), so this content class was never hit.

## dosu-bot comment structure (domain knowledge)

A dosu comment has a fixed 3-part shape, each marked by an HTML comment:
- `<!-- Greeting -->` + `Hi @user! I'm [Dosu](https://go.dosu.dev/dosubot) and I'm helping the <X> team.`
- `<!-- Answer -->` + the actual technical answer (CONTENT — keep; often carries `[[N]](github /blob/ url)` source-citation links).
- `<!-- Dosu Comment Footer -->` + `To reply, just mention @dosu` + `Docs are dead. Just use Dosu` + `Share context across your team and agents. Try Dosu` + a shields.io/camo badge line (the no-space chunk-breaker).

**Key complication:** when a comment is **blockquoted** (`> `) in a reply or **email-rendered** (user answered the bot via email; GitHub captures the email with `发件人:`/`主题:` headers), the HTML markers vanish but the text/badges remain. Marker-anchored strips leak these.

Round 3 = anchor fix on #9: the plain email taglines (`Docs are dead. Just use Dosu.`, no link) failed the original `dosu.dev`-domain anchor → relaxed to the `Dosu` brand token.

## Content-safety methodology (the core discipline)

Every new text-phrase strip was gated by: **grep the whole corpus — does the phrase ever appear embedded in user prose?** If never (always line-leading bot boilerplate) → safe. Plus a dual anchor (exact phrase + a dosu/brand token) to make false-positives impossible.

**Deliberate rejections (would eat real content):**
- **Email-quote headers** (`发件人: dosubot[bot]`, `主题: Re: …`) in MinerU__4878 — NOT stripped: the `主题:` line IS the question and the quoted body IS the dosu answer. Stripping the email structure would destroy content. Left the header remnants as accepted noise in 1 file.
- **User-quoted greetings** (`> **@eziohzy**: > Hi @…I'm Dosu…`) — preserved: the markerless-greeting anchor requires the line to START WITH `Hi @`/`你好@` after prefix-strip; a user-attribution prefix means it's preserved.
- **`![]()` syntax example** in a code span (MinerU__4986, someone explaining markdown image syntax) — preserved by requiring a **non-empty URL** (`![…](…)` → `[^)]+`).

## Architecture decisions

- **Pure module `src/github/discussion_cleaning.py`** (only `import re`, mcp-free) — single source for `strip_noise`; `index_discussions.py` imports it. Rationale: the dev validation script was inline-copying the logic → drift risk.
- **dev-import-hook constraint:** `block_dev_imports_src.py` forbids `from src.` in `dev/` files — an INTENTIONAL project convention ("intentional duplication, not drift", documented in `dev/repo_exploration/DOCS.md`). So the dev re-clean/validation scripts keep **inline copies** with a source-of-truth header comment, NOT imports. This reversed an initial (wrong) plan to import from src into dev.
- **Shared `src/github/text_cleaning.py`** — the GENERIC primitives (any-`<img>` strip, any-`![](url)` markdown-image strip, base64 data-URI strip, failed-upload, 1000-char no-space net) extracted so BOTH `discussion_cleaning.py` AND `index_issues.py` use one copy. dosu-specific strips stay in discussion_cleaning. Refactor guarded by SHA256 byte-identity over the 78-file discussions corpus.

## Issues extension

`index_issues` already drops whole **bot comments** (`strip_comments_noise`: Author ends `[bot]`) → dosu footers/badges never reach the issue MDs (shields/camo = 0). But it had **no image handling and no no-space net**. Added `strip_generic_noise(body)` + `strip_generic_noise(comments)` additively.

**Regex-too-strict finding** (surfaced by the shared module): the image regexes only matched `<img width=… height=… alt=… src=…>` in exact order and markdown images **with** a file extension. Issues had 17 `<img>` without `height` + 12 `![image](…/assets/<uuid>)` without extension = **29 images leaking**. Loosened to `<img\b[^>]*>` (any tag) + `![…](…)` (any markdown image) — content-safe (images carry no searchable text) and robustifies both pipelines. Discussions corpus had 0 survivors (their screenshots all had width+height), so no discussion re-clean needed for this.

## Re-clean of existing corpora

Both corpora were re-cleaned in place (the pipeline only auto-cleans FUTURE fetches). Mechanics:
- **Backup-before-overwrite** — the corpora (`rag-cli/data/documents/github_*`) are gitignored → no git safety net.
- **strip_noise / strip_generic_noise WITHOUT the format pass** — the built MDs already have `# title` (H1) and no metadata block; re-running the full `strip_discussion_noise` would mis-fire its title-extraction and **eat the first `## ` body heading** (13 discussion files have real `## ` headings). The re-clean applies only the noise part.
- **rstrip change-detection** — only rewrite files whose content differs beyond trailing whitespace (avoided 19→0 spurious 1-char `\n\n`→`\n` rewrites + re-index churn).
